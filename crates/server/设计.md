# Server 设计

## 定位

`server` 是运行在电脑上的本地控制服务。

职责：

- 接收动作请求
- 执行系统输入

不负责：

- 前端托管
- 客户端 UI
- 配对码
- 设备发现
- 云端中继

---

## 动作范围

当前只支持三类动作：

1. `send-key`
2. `send-shortcut`
3. `paste-text`

具体能力：

- `send-key`
  - `tab`
  - `enter`
  - `backspace`
- `send-shortcut`
  - `ctrl-c`
  - `ctrl-v`
  - `shift-tab`
- `paste-text`
  - 任意文本
  - 包括 `"\n"`

说明：

- `send-key: enter` 表示发送一次 Enter 按键
- `paste-text: "\n"` 表示粘贴一个换行字符

两者语义不同。

---

## API

唯一控制接口：

```http
POST /api/action
```

健康检查：

```http
GET /health
```

`/health` 不认证，返回：

```text
ok
```

---

## 请求模型

Rust 内部模型：

```rust
enum Action {
    SendKey { key: KeyName },
    SendShortcut { shortcut: Shortcut },
    PasteText { text: String },
}

enum KeyName {
    Tab,
    Enter,
    Backspace,
}

enum Shortcut {
    CtrlC,
    CtrlV,
    ShiftTab,
}
```

JSON 示例：

发送 Tab：

```json
{
  "action": {
    "type": "send-key",
    "key": "tab"
  }
}
```

发送 Enter：

```json
{
  "action": {
    "type": "send-key",
    "key": "enter"
  }
}
```

发送 Backspace：

```json
{
  "action": {
    "type": "send-key",
    "key": "backspace"
  }
}
```

发送 `ctrl-c`：

```json
{
  "action": {
    "type": "send-shortcut",
    "shortcut": "ctrl-c"
  }
}
```

发送 `ctrl-v`：

```json
{
  "action": {
    "type": "send-shortcut",
    "shortcut": "ctrl-v"
  }
}
```

发送 `shift-tab`：

```json
{
  "action": {
    "type": "send-shortcut",
    "shortcut": "shift-tab"
  }
}
```

粘贴文本：

```json
{
  "action": {
    "type": "paste-text",
    "text": "你好，世界"
  }
}
```

粘贴换行：

```json
{
  "action": {
    "type": "paste-text",
    "text": "\n"
  }
}
```

## 认证

受保护接口统一使用 Bearer token。

请求头：

```http
Authorization: Bearer <token>
```

规则：

- `/api/action` 必须认证
- `/health` 不认证

认证失败返回：

```http
401 Unauthorized
```

错误文本：

```text
missing or invalid authorization token
```

## 配置

配置项：

- `host`
- `port`
- `auth_token`

优先级：

```text
CLI > ENV > Config File > Default
```

实现方式：

- `clap` 负责 CLI 参数解析
- `config` 负责默认值、环境变量、配置文件加载与合并
- CLI 覆盖最终配置

默认值：

- `host = 127.0.0.1`
- `port = 8765`
- `auth_token` 无默认值

如果未提供 `auth_token`，服务直接拒绝启动。

---

## 错误语义

- `400 Bad Request`
  - 请求结构非法
  - 参数缺失
  - `text` 为空
  - 不支持的 `key`
  - 不支持的 `shortcut`
- `401 Unauthorized`
  - 缺少 token
  - token 错误
- `500 Internal Server Error`
  - 剪贴板失败
  - 输入注入失败
  - 其他本地执行失败

---

## 非目标

- 配对码
- 客户端设计
- 设备发现
- 多设备授权列表
- 账号体系
- 云端中继

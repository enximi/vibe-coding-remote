# 产品文档

本目录用于存放产品需求、功能定义、交互说明、技术设计、验收标准与实施计划等文档。

## 文档列表

- `prd.md`
  记录产品为什么存在、要解决什么问题、面向谁、目标是什么、不打算做什么。
  这份文档主要回答“这个产品到底要做什么”。

- `feature-list.md`
  记录当前产品有哪些功能、这些功能如何分组、哪些是核心功能、哪些是支撑功能、哪些还没做。
  这份文档主要回答“这个产品现在具体有哪些功能”。

- `ux-spec.md`
  记录用户能看到什么、每个页面或入口怎么进入、每个按钮有什么作用、不同状态下界面和交互如何变化。
  这份文档主要回答“用户实际会怎么使用它、会感知到什么”。

- `technical-design.md`
  记录功能在技术上如何实现、模块怎么拆、状态如何流动、接口如何设计、关键实现方案和取舍是什么。
  这份文档主要回答“这个产品是怎么实现的，为什么这样实现”。

- `acceptance-criteria.md`
  记录每个功能做到什么程度才算完成、需要覆盖哪些正常场景和异常场景、验收时重点检查什么。
  这份文档主要回答“什么叫做做完了，什么叫做符合预期”。

- `implementation-plan.md`
  记录当前阶段目标、里程碑、任务拆解、依赖关系、风险项、验证计划和进度记录。
  这份文档主要回答“接下来按什么顺序做、当前做到哪里了”。

## 使用建议

- 当你在想“这个需求该不该做、为什么做”时，先写 `prd.md`
- 当你在想“这个产品现在到底有哪些功能”时，先写 `feature-list.md`
- 当你在想“按钮和页面具体应该怎么表现”时，先写 `ux-spec.md`
- 当你在想“代码和接口应该怎么设计”时，先写 `technical-design.md`
- 当你在想“AI 或自己有没有把功能做歪”时，先看 `acceptance-criteria.md`
- 当你在想“下一步先做什么、怎么拆任务”时，先写 `implementation-plan.md`
